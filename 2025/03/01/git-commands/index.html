Git指令集

2025 年 3 月 1 日 发布在 工具

  1. mkdir

    创建目录

  2. cd <目录名>

    进入目录

  3. pwd

    显示工作目录

  4. git commit -m""

    查看修改内容

  5. git add <文件名/目录名>

    添加至暂存区

  6. git status

    查看状态

  7. git reset --hard Head()

    版本回退

    Head后加^为上个
^^为上上个
~n为上n个

  8. git reset --hard ()

    版本回退

    hard 后加版本号

  9. cat <文件名>

    查看文件内容

  10. git reflog

    看版本log

  11. git log

    查看提交历史

  12. git log --pretty=online

    查看美化版提交历史

  13. git checkout -- <文件名>

    丢弃工作区修改（恢复删除）

  14. git reset HEAD <文件名>

    撤销暂存区修改，放回工作区

  15. git diff

查看修改内容

  1. rm <文件名>（rm -r 可删除目录）

删除文件

  1. git branch -m <旧名字> <新名字>

分支改名

  1. git branch

查看分支

  1. git push origin <仓库名>

推送<仓库名>仓库到网络

红色字体部分已经落后于版本，推荐使用switch。

  1. git checkout <目录名>

切换到<目录名>目录

  1. git checkout -b <分支名>

创建并且换到名为<分支名>的分支

  1. git checkout -

切换到资金使用的分支

  1. git checkout <分支名> && git commit -m "内容"

切换到<分支名>，同时提交当前目录的修改

  1. git merge <分支名>

合并<分支名>分支到当前分支

  1. git branch -d <分支名>

删除<分支名>分支

  1. git switch <分支名>

切换到<分支名>分支（新版）

  1. git switch -c <分支名>

在该分支下创建并切换到<分支名>分支

  1. 两个分支都更改下无法合并，需手动修改后提交，在一个分支中合并分支不会同化另一个分支内容，而是使suo'zai'fensuozaifen

  2. git log --graph

  3. git merge --no-ff -m"" <>

  4. git add -f

  5. git check-ignore -v

  6. git diff
